# QA Spec Writing Guidance

Purpose
- Define test strategy, acceptance criteria, and validation scope.

When to Use
- A user requests testing plans, acceptance criteria, or QA workflows.

Process
- Tie tests to product requirements and risk areas.
- Use clear naming conventions and explicit pass/fail conditions.

Output Rules
- Include smoke, regression, and edge-case coverage.
- List required environments, data, and tooling.
- Flag any gaps or missing inputs.
