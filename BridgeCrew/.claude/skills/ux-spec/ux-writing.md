# UX Spec Writing Guidance

Purpose
- Produce UX flow docs, wireframe notes, and accessibility requirements.

When to Use
- A user requests UX flows, wireframes, or usability requirements.

Process
- Keep flows structured as steps with decisions and state transitions.
- Align accessibility guidance with product constraints and platforms.
- Reference product requirements to avoid scope drift.

Output Rules
- Use consistent step numbering for flows.
- Include assumptions and edge cases.
- Prefer short annotations over long prose.
